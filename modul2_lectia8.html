Securitatea în PHP

Acasă | Cursuri | Modul 2

Principale riscuri de securitate în PHP

  • Injecții SQL
  • Cross-Site Scripting (XSS)
  • Cross-Site Request Forgery (CSRF)
  • Injecții de cod (Code Injection)

Prevenirea injecțiilor SQL

Folosește întotdeauna prepared statements cu PDO sau MySQLi:

<?php
$stmt = $pdo->prepare("SELECT * FROM utilizatori WHERE email = :email");
$stmt->execute(['email' => $email]);
$user = $stmt->fetch();
?>

Prevenirea XSS

Escapează datele afișate în HTML cu htmlspecialchars():

<?php
echo htmlspecialchars($user_input, ENT_QUOTES, 'UTF-8');
?>

Prevenirea CSRF

Folosește tokeni CSRF în formulare și verifică-i la trimitere.
